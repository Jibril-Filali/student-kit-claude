Questions fréquentes

5 thèmes · ~4 min · Les vraies questions que tout le monde se pose

01 · Ethique et usage scolaire

Les questions que tout étudiant se pose avant de se lancer.

?

Est-ce que c'est de la triche ?

Ca dépend de comment tu l'utilises et de la politique de ton établissement.

  • Pas de la triche : utiliser Claude pour comprendre un concept, déboguer une erreur, améliorer la qualité de ton code, apprendre une technologie.
  • Potentiellement problématique : copier-coller une réponse sans la comprendre et la rendre comme ton travail lors d'une évaluation qui l'interdit.
De plus en plus de profs intègrent l'IA dans leur pédagogie. Certains l'autorisent, d'autres non. En cas de doute, demande directement à ton prof avant l'évaluation.
?

Mon prof peut-il détecter que j'ai utilisé Claude ?

Pour du code : pratiquement non. Les détecteurs d'IA ont un taux de faux positifs très élevé sur le code — ils accusent souvent du code humain d'être généré par IA.

Pour du texte rédigé : les détecteurs (GPTZero, Turnitin) sont plus sensibles, mais restent peu fiables. Un texte entièrement généré et non retravaillé peut être repéré.

Ce n'est pas la bonne question à se poser. Utilise Claude pour apprendre, pas pour contourner. Si tu copies sans comprendre, les lacunes remonteront à la surface lors des examens sur table.
?

Est-ce que ça va me rendre dépendant(e) ?

Si tu copies sans comprendre : oui, tu creuses des lacunes. Si tu utilises Claude pour apprendre plus vite, non.

Bonne approche

Demande toujours "pourquoi" en plus du "quoi". Fais relire ton code avec une explication de chaque changement. Utilise le sous-agent tutor pour comprendre les concepts.

Mauvaise approche

Copier le code sans lire. Accepter la première réponse sans tester. Utiliser Claude pour les évaluations sur table que tu ne peux pas passer sans lui.

02 · Coût et accès

Ce que tu paies, ce que tu obtiens.

?

C'est gratuit ?

Claude.ai (la version web) a une offre gratuite avec des limitations : modèle moins puissant, nombre de messages limité par jour.

Claude Code (le CLI de ce guide) nécessite un abonnement Claude Pro (~20$/mois) ou des crédits API. L'abonnement Pro couvre les deux : Claude.ai Pro et Claude Code.

Il existe aussi une option crédits API (pay-as-you-go) via console.anthropic.com si tu veux payer uniquement ce que tu utilises sans abonnement mensuel.
?

L'abonnement Pro vaut-il le coup pour un étudiant ?

Si tu utilises Claude Code régulièrement pour tes projets de développement, oui. Un bug résolu en 5 minutes au lieu de 2 heures, multiplié par ta semaine de cours, dépasse largement 20$.

Si tu veux juste tester, commence avec les crédits API ou la version gratuite de Claude.ai. Claude Code lui-même a aussi une période d'essai.

?

Quelle différence entre abonnement Pro et crédits API ?

Abonnement Pro (~20$/mois)

Accès illimité à Claude.ai + Claude Code. Idéal si tu utilises Claude tous les jours. Prix fixe, pas de surprise.

Crédits API (pay-as-you-go)

Tu paies au token utilisé. Idéal si ton usage est irrégulier. Peut coûter moins cher si tu utilises peu, plus si tu utilises beaucoup.

03 · Confidentialité

Ce qu'Anthropic voit, ce qu'il ne voit pas.

?

Mes conversations sont-elles privées ?

Anthropic peut traiter les conversations pour améliorer ses modèles, sauf si tu désactives cette option dans les paramètres de ton compte. Les données sont hébergées aux Etats-Unis.

Pour un usage scolaire normal (projets de cours, exercices Java, requêtes SQL), ce n'est pas un problème. Tes conversations de TP ne présentent aucun risque particulier.

?

Mon code est-il en sécurité ?

Tes fichiers restent sur ton PC. Claude Code transmet uniquement le contenu des fichiers qu'il lit aux serveurs Anthropic pour générer une réponse. Aucune copie permanente n'est stockée.

Ne partage jamais de code contenant des clés API, mots de passe, tokens ou données personnelles dans une conversation Claude. Prends l'habitude de ne jamais coller de secrets dans n'importe quel outil en ligne.
?

Anthropic peut-il voir mes fichiers de projet ?

Uniquement les parties que Claude lit pour répondre à ta question. Si tu demandes à Claude de corriger Main.java, le contenu de ce fichier est transmis. Les autres fichiers de ton projet ne sont pas envoyés automatiquement.

04 · Utilisation pratique

Les questions sur l'usage au quotidien.

?

Quelle différence entre Claude.ai et Claude Code ?

Claude.ai (Web)

Interface web dans le navigateur. Tu poses des questions, tu obtiens des réponses. Pas d'accès direct à tes fichiers. Idéal pour les explications, les documents, les questions générales.

Claude Code (CLI)

Outil terminal qui accède directement à tes fichiers. Il lit ton code, modifie tes fichiers, lance des commandes. Beaucoup plus puissant pour le développement. C'est ce que ce guide couvre.

?

Ca marche pour Java, MySQL, MongoDB ?

Oui, et très bien. C'est exactement le stack pour lequel ce kit est conçu.

  • Java : gestion des exceptions, JDBC, Maven, héritage, génériques, streams, Javadoc.
  • MySQL : requêtes complexes, JOIN, GROUP BY, optimisation, PreparedStatement, index.
  • MongoDB : agrégations, pipelines, schémas, drivers Java, find() vs aggregate().
?

Ca marche si mon prof n'utilise pas Claude ?

Oui. Claude Code est un outil local sur ton PC. Ton prof n'a aucune interaction avec lui, il ne sait pas que tu l'utilises (sauf si tu lui en parles). Tu l'utilises comme n'importe quel autre outil : un compilateur, un debugger, Stack Overflow.

?

Ca marche pour les TP de groupe ?

Oui. Chaque membre installe Claude Code sur son PC. Claude aide chacun individuellement — il lit les fichiers du projet en local. Pour les projets partagés via Git, chacun l'utilise de son côté sur sa copie locale.

05 · Questions techniques

Installation, compatibilité, prérequis.

?

Ca marche sur Mac et Linux ?

Oui. Le CLI claude est identique sur tous les systèmes. Ce guide est orienté Windows/PowerShell, mais Claude Code fonctionne parfaitement sur macOS (Terminal, zsh) et Linux (bash).

Seules les commandes PowerShell (nouveau-projet, update-kit...) sont spécifiques à Windows. Sur Mac/Linux, tu peux créer des équivalents en bash ou utiliser directement Claude Code sans ces raccourcis.

?

J'ai besoin d'internet ?

Oui. Claude Code communique avec les serveurs Anthropic pour générer ses réponses. Il ne fonctionne pas hors ligne. Une connexion standard suffit, pas besoin d'une connexion rapide.

?

Quels sont les prérequis ?

  • Node.js LTS (pour installer Claude Code via npm)
  • Un compte Anthropic avec abonnement Pro ou crédits API
  • PowerShell (déjà installé sur Windows 10/11)
  • Un terminal : PowerShell, VS Code terminal, ou autre

Pas besoin de Java, Maven ou autre pour Claude Code lui-même. Tu les as sûrement déjà si tu fais de la programmation en cours.

?

Claude peut-il lancer mes programmes Java ?

Oui, si Java et Maven sont dans ton PATH. Claude Code peut exécuter mvn compile, mvn test, javac, etc. Le settings.json du kit pré-autorise ces commandes pour éviter les confirmations répétées.

Une question pas dans la liste ? Utilise le terminal IA en bas à droite de cette page, ou pose-la directement à Claude avec claude dans ton terminal.
